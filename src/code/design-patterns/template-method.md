> Базовый класс задаёт скелет алгоритма, потомки переопределяют шаги.

## Где помогает
- Есть общий порядок действий, детали различаются (парсеры, экспортёры).
- Требуется гарантировать последовательность шагов.

## Мини-пример
```ts
abstract class Parser{
  parse(){ this.read(); this.transform(); }
  abstract read():void; abstract transform():void;
}
class JsonParser extends Parser{
  read(){ /* ... */ } transform(){ /* ... */ }
}
new JsonParser().parse();
```

## Замечания
- Можно добавлять `hook`-методы (по умолчанию пустые).
- Перегрузка слишком многих шагов делает базу хрупкой — держите скелет коротким.

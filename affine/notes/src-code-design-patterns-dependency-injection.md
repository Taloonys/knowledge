> Объект получает зависимости извне, сам их не создаёт.

## Где помогает
- Тестирование (подставить моки), переиспользование и конфигурируемость.
- Разрывает жёсткие связи с конкретными реализациями.

## Мини-пример
```ts
class Service{
  constructor(private repo:{all():any[]}){}
  list(){ return this.repo.all(); }
}
const svc = new Service(new UserRepo());
```

## Замечания
- Три способа: через конструктор, сеттер, метод.
- Контейнер DI полезен, но не обязателен; избегайте магии, делайте связи явными.
